ArcaNet

A phrase-level combinatorial composer, in homage to Kircher's Arca Musarithmica.

Overview Audio samples Code walkthrough GitHub ↗

“Out of a few phrases, an almost unbounded number of melodies.” — after Athanasius Kircher, Musurgia Universalis, 1650

What it is

ArcaNet is the reference prototype for a position paper submitted to Computer Music Journal arguing that phrase-level combinatorial generation is a more fruitful — and more musically defensible — direction than autoregressive end-to-end neural models for symbolic music. Its smallest generative unit is a phrase (a one- or two-bar figure with a definite harmonic and rhythmic identity), not a token. Every compositional decision is inspectable, reversible, and editable.

Why phrases?

Token-level autoregressive models optimize a next-event likelihood that has no musical meaning: a “good” sequence in MIDI-event space is not the same as a good musical phrase. Kircher's 1650 Arca already showed that a modest bank of well-formed phrases, recombined under grammatical rules, produces a combinatorial explosion of musically coherent output. ArcaNet re-casts that insight with a neural-style selector over a curated bank of phrases, plus a hard constraint layer that enforces common-practice harmonic grammar and stepwise voice leading.

What's in the box

~550 LOC Python 3.9+ No dependencies Deterministic

The core arcanet.py uses only the Python standard library. It ships with a phrase bank, a softmax-with-temperature selector, a grammar/voice-leading constraint layer, and a raw-bytes Standard MIDI File writer. The full selection trace (top-3 candidates per slot with probabilities) is emitted to a human-readable log at every run.

Run it

git clone https://github.com/your-github-username/arcanet.git
cd arcanet
python arcanet.py

With the default seed (2026) the command produces three deterministic MIDI files, a selection trace, and a statistics JSON. The outputs are byte-identical across runs so the paper's claims can be verified mechanically.

Listen first

You can hear the three default compositions on the audio samples page — they are rendered with a tiny additive-synthesis piano voice so no soundfont is required. For a faithful rendering, open the MIDI files in any DAW with a real piano instrument.
