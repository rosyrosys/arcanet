Full source: arcanet.py (~550 lines, standard library only). The snippets below show the six load-bearing pieces; everything else is plumbing.

1. Phrase, the smallest compositional unit

A phrase carries a chord, a harmonic function, a role, a density score, explicit first- and last-pitches (for voice-leading computation), and the list of note events. Every field is cheap to inspect; the selector will feature-extract from them directly. 

@dataclass(frozen=True)
class Phrase:
    pid:         str
    chord:       str            # "I", "ii", "V", ...
    function:    str            # "T", "PD", "D"
    role:        str            # "opener", "answer", "middle", "close"
    density:     float          # notes / beat
    first_pitch: int
    last_pitch:  int
    notes:       tuple          # ((pitch, onset, dur, vel), ...)
    total_beats: float

2. A curated phrase bank

The bank is explicit, small, and hand-readable — closer to Kircher's rows of pre-composed fragments than to an opaque learned embedding. Add phrases by appending literals; the selector adapts automatically. 

BANK: list[Phrase] = [
    Phrase("I_opener_01", "I", "T", "opener", 1.25, 60, 67,
           notes=((60, 0.0, 1.0, 90), (64, 1.0, 1.0, 82),
                  (67, 2.0, 2.0, 88)), total_beats=4.0),
    Phrase("V_close_01", "V", "D", "close",   1.5,  71, 72,
           notes=((67, 0.0, 0.5, 84), (71, 0.5, 0.5, 80),
                  (74, 1.0, 1.0, 90), (72, 2.0, 2.0, 78)),
           total_beats=4.0),
    # ... ~30 more phrases covering all chords and roles ...
]

3. The neural-style selector

A scoring function maps (candidate, previous, role) to a scalar; a softmax with temperature turns those scores into a distribution; we sample, and log the top three competitors for replay and inspection. 

def score(cand: Phrase, prev: Phrase | None, role: str) -> float:
    s  = role_bias(cand.role, role)
    s += density_preference(cand.density, role)
    if prev is not None:
        s -= 0.18 * abs(cand.first_pitch - prev.last_pitch)   # voice-lead
        s += function_transition_bonus(prev.function, cand.function)
    return s

def select(prev, role, rng, temperature=0.9):
    pool = [c for c in BANK if obeys_grammar(prev, c) and jump_ok(prev, c)]
    logits = [score(c, prev, role) / temperature for c in pool]
    probs  = softmax(logits)
    pick   = rng.choices(pool, weights=probs, k=1)[0]
    top3   = sorted(zip([c.pid for c in pool], probs), key=lambda x: -x[1])[:3]
    return pick, top3

4. The hard constraint layer

Grammar and voice-leading are not soft preferences — they are enforced by filtering the candidate pool before softmax. This is the layer that makes bad moves impossible rather than merely improbable. 

def obeys_grammar(prev: Phrase | None, cand: Phrase) -> bool:
    if prev is None:
        return True
    return cand.function in FUNCTION_TRANSITIONS[prev.function]

def jump_ok(prev: Phrase | None, cand: Phrase, max_semitones: int = 5) -> bool:
    if prev is None:
        return True
    return abs(cand.first_pitch - prev.last_pitch) <= max_semitones

5. Form templates bias role distribution

A form template is a sequence of slot roles. Plug in AABA, antecedent-consequent, or a free chain; the selector does the rest. 

FORMS = {
    "AABA_16":   ["opener", "answer", "opener", "answer",
                  "middle", "middle", "opener", "close"],
    "sentence_8": ["opener", "opener", "middle", "close"],
    "free_14":   ["opener"] + ["middle"] * 5 + ["close"],
}

6. Standard MIDI File output, no third-party library

The MIDI writer is raw bytes: a header chunk, a tempo meta-event, a 4/4 time signature, and delta-time-encoded note-on/note-off events. ~40 lines total, no mido, no pretty_midi. 

def composition_to_midi(comp, path, tempo_bpm=92, ppq=480):
    events = [(0, tempo_meta(tempo_bpm)), (0, time_sig_meta(4, 4))]
    offset = 0.0
    for ph in comp.phrases:
        for (pitch, onset, dur, vel) in ph.notes:
            on  = int(round((offset + onset)       * ppq))
            off = int(round((offset + onset + dur) * ppq))
            events.append((on,  bytes([0x90, pitch, vel])))
            events.append((off, bytes([0x80, pitch, 0])))
        offset += ph.total_beats
    write_smf(path, events, ppq)

Why this matters

Every one of these snippets is six lines or fewer. A musician who does not program for a living can still read every phrase in the bank, every scoring term, every grammar transition. This is the whole point: PLCA is not designed to outperform a large language model on perplexity — it is designed to give composers an inspectable, editable, and musically grounded generative system to collaborate with.
